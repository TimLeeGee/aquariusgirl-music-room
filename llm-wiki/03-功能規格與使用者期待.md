# 功能規格與使用者期待

#spec

| 範圍 | 正確行為 | 錯誤行為 | 來源 |
| --- | --- | --- | --- |
| 音樂來源 | 僅載入使用者選擇的本機音樂 | 串流、下載、偷掃整台硬碟 | README.md、AGENTS.md |
| 音樂資料保存 | 保存 metadata、播放清單、設定 | 保存音樂檔本體、File、Blob、ArrayBuffer、object URL | AGENTS.md |
| 封面/資訊寫回 | 一次寫入原始檔、重新讀回，readback hash 驗證後再更新單曲 IndexedDB | 只憑前端 draft 或 IPC 成功就顯示成功；雙保存路徑 | AGENTS.md、release-delivery/README.md |
| 播放中寫回 | 只暫停並卸下目前目標歌曲的 audio src，完成後恢復位置與播放狀態 | 重掃曲庫、全庫資料庫重寫，或讓 Windows 鎖檔無明確錯誤 | AGENTS.md |
| 模式切換 | 一般、Mini、OBS 共用同一 audio DOM 節點，切換連續播放 | 包裝 audioElement 造成 React 卸載重建，之後把所有錯誤說成阻擋播放 | README.md、docs/releases/0.1.49-checksums.md |
| 清單排序 | 一般播放清單保存 trackIds；全部歌曲只保存被移動曲目的 addedAt | 自訂排序時全庫 rewrite | AGENTS.md |
| AI | 回答及建立清單只使用已索引歌曲的安全 metadata | 編造未載入歌曲、檔案路徑或清單內容 | AGENTS.md、docs/AI_PLAYLIST_AUDIT_0.1.16.md |
| UI 自訂 | 文案可覆寫、角色中英文名稱全站套用 | 期待 runtime 改到 OS 視窗或 dock product name | docs/releases/0.1.47-checksums.md、docs/releases/0.1.48-checksums.md |
| Dev 播放驗證 | electron:dev 用拖曳 blob 驗證播放 | 把 http origin 無法載入 file:// 音源當作產品 bug | docs/releases/0.1.49-checksums.md |
