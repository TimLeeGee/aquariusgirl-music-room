# 常用提示詞

#prompt

| 情境 | 可複製提示詞 |
| --- | --- |
| 新功能 | 請先讀 llm-wiki 的 00、01、02、03、05、06、07 與相關來源。不要修改程式；先提出最小方案、涉及檔案、風險和驗收。 |
| 修 bug | 請先讀 llm-wiki 的 04、05、06、07、09 和相關 source。先追根因、列所有 callers 與回歸風險；不要修改，直到我明確說可以修改。 |
| UI 變更 | 請先讀 llm-wiki 的 02、03、04、05、06、07。保持既有版面與大曲庫捲動界線，先以可觀察行為描述變更與驗收。 |
| 歌曲資訊/封面 | 請遵守 llm-wiki 的 03、04、05、06、09；用暫存音樂複本和隔離 profile，不動真實 Music 資料夾，並報告 readback hash 證據。 |
| 驗收 | 請依 llm-wiki 的 06 設計最小驗收。逐項列出實際跑過的命令、結果、未跑項目與 packaged/Windows 限制。 |
| 維護 Wiki | 請根據實際 diff 與原始文件更新受影響 llm-wiki 頁面，保留來源、衝突與待確認項，最後在 99-更新日誌加一筆。 |
| 發布前 | 請讀 llm-wiki 的 06、08、12。先盤點版本與工作樹，再列精準 stage 清單、文件同步清單、打包/雜湊/遠端讀回證據；未獲授權不要 push。 |
