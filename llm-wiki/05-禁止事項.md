# 禁止事項

#project #validation

| 禁止事項 | 原因 | 來源 |
| --- | --- | --- |
| 不使用 YouTube、串流、線上音樂、下載或生成圖片 | 產品是本地優先播放器 | AGENTS.md、README.md |
| 不掃描整台硬碟 | 只能處理使用者明確選取的本機音樂 | AGENTS.md |
| 不把音樂本體或 browser binary 資料寫進 localStorage/IndexedDB | 保存範圍只限 metadata、清單和設定 | AGENTS.md |
| 不為小改動新增套件、抽象或全庫工作 | 維持最小可驗證變更與大曲庫效能 | AGENTS.md |
| 不清 IndexedDB、不重掃曲庫、不全庫 rewrite 來處理單曲 metadata/cover 問題 | 避免資料損失與性能回歸 | AGENTS.md |
| metadata/cover-only 更新不可改 localUrl 或 mediaVersion，不可造成同來源 audio.load() | 防止播放中斷 | AGENTS.md |
| 不回到播放器本地保存與原始檔寫回的雙路徑 | 原始檔寫回、讀回與單曲保存是唯一線 | AGENTS.md |
| 不再新增 renderer window.confirm/window.alert | Windows Electron 關閉同步 dialog 後會損壞文字輸入與 native select 焦點 | AGENTS.md |
| 不改變 audioElement 在三模式中的 reconcile 位置 | 會導致 React 卸載重建與播放中斷 | AGENTS.md |
| 不讓大量 metadata 逐首全庫 map、setState 或 DB 寫入 | 會讓大曲庫退化並製造大量 commits | docs/releases/0.1.51-checksums.md |
| 不只靠時間節流冒充大曲庫 commit 上限 | 無法保證批數上限，必須有實際 batch 與 check | docs/releases/0.1.51-checksums.md |
| 不在 node strip-types 直接載入檔加入無副檔名 runtime value import | aiTrackSearch.ts 與 trackDisplay.ts 的 check 會失敗 | AGENTS.md |
| 不把 installer、GGUF、大型產物、憑證或私鑰 commit | 公開 repo 與交付安全邊界 | release-delivery/README.md、[[08-GitHub發布守門員]] |
| 未獲明確指示不 commit、push、tag、發布或重打 installer | 外部狀態和發行範圍須由使用者授權 | AGENTS.md |

## 不確定時的處理

- [ ] 先查 [[12-來源文件索引]] 和最新 source，不以舊 release note 推論目前行為。
- [ ] 若要突破任何限制，列出影響、替代方案與需要的使用者確認。
- [ ] 若資料安全、寫回、跨平台或打包結論不完整，標示限制而非宣稱通過。
